快速看懂兩種 API

同一個功能，兩種寫法

這一頁只做對比，不當正式主線。比賽實作請優先看組合式 API。

對比重點：資料放哪裡、方法放哪裡、怎麼回傳、哪裡會看到 this。
比較項目 選項式 API 組合式 API
資料 `data()` 裡 return `setup()` 裡用 `ref()`
方法 `methods` `setup()` 裡寫 function
畫面拿資料 Vue 自動整理 靠 `return` 交出去
`this` 常見 通常不用

選項式 API

{{ msg }}

Vue.createApp({
  data() {
    return {
      msg: '早安，這是選項式 API'
    }
  },
  methods: {
    changeMsg() {
      this.msg = '這裡會用到 this'
    }
  }
}).mount('#app-option')

組合式 API

{{ msg }}

const { createApp, ref } = Vue

createApp({
  setup() {
    const msg = ref('早安，這是組合式 API')

    function changeMsg() {
      msg.value = '這裡直接改 ref'
    }

    return { msg, changeMsg }
  }
}).mount('#app-setup')
